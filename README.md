# 🌱 EcoTrack — Carbon Footprint Awareness Platform

EcoTrack helps individuals **understand, track, and reduce** their carbon
footprint through a few simple questions and personalized, actionable insights.

Answer a short questionnaire about your everyday life and EcoTrack will:

1. **Understand** — estimate your annual carbon footprint from transport, home
   energy, diet, and consumer goods using published emission factors.
2. **Track** — store each calculation privately on your device and chart your
   progress over time against a personal reduction goal.
3. **Reduce** — generate prioritized, plain-language actions (AI-personalized
   with Google Gemini, with a built-in fallback engine) that target your biggest
   sources first and estimate the CO₂e each action could save.

## Getting started

### Prerequisites

- Node.js 18.18+ (Node 20+ recommended)

### Install & run

```bash
npm install
npm run dev
# open http://localhost:3000
```

The app is **fully functional with no API key** — it uses a deterministic
rule-based insights engine as a fallback.

### Enable AI-personalized insights (optional)

1. Get a free Gemini API key from
   [Google AI Studio](https://aistudio.google.com/apikey).
2. Copy the example env file and add your key:

   ```bash
   cp .env.example .env.local
   # then set GEMINI_API_KEY=your_key_here
   ```

3. Restart `npm run dev`. Insights are now generated by Gemini, with automatic
   fallback to the rules engine if the API is unavailable.

> The key is read only on the server in `app/api/insights/route.ts` and is never
> exposed to the browser.

## Architecture

```
app/
  layout.tsx            Root layout, fonts, metadata, skip link
  page.tsx              Main application (client orchestration)
  globals.css           Tailwind + base/a11y styles
  api/insights/route.ts Secure POST endpoint (validation, rate limit, AI+fallback)
components/
  ui/                   Reusable, accessible form controls
  *.tsx                 Feature components (summary, breakdown, insights, goal…)
lib/
  carbon/               Domain: emission factors, types, pure calculator, schema
  insights/             Rule-based engine, AI client, prompt, response schema
  storage.ts            Validated, privacy-first localStorage layer
  rate-limit.ts         In-memory fixed-window limiter
  format.ts             Locale-aware formatting
```

### Data & methodology

Emission factors are documented inline in `lib/carbon/factors.ts` with citations
(UK DEFRA, US EPA, and peer-reviewed dietary studies). Figures are global-average
approximations intended for **awareness and education**, not regulatory
reporting. All user data stays on the device — there is no account and no server
database.

## 🤖 AI tools & prompts

This section documents, for full transparency, **(A)** the AI used *inside* the
product and **(B)** the AI tooling used *to build* the project.

### A. AI used in the product — Google Gemini

| Aspect | Detail |
| --- | --- |
| **Provider / tool** | Google **Gemini** via the official `@google/generative-ai` SDK |
| **Model** | `gemini-2.5-flash` (override with the `GEMINI_MODEL` env var) |
| **Where it runs** | Server-only: `app/api/insights/route.ts` → `lib/insights/ai.ts` (guarded by `import "server-only"`, so the key never reaches the browser) |
| **Generation config** | `temperature: 0.4`, `responseMimeType: "application/json"` |
| **Input sent** | Only anonymous lifestyle figures + the computed footprint breakdown — no names, accounts, or device identifiers |
| **Output handling** | Response is parsed by `extractJson()` then **validated with a Zod schema** (`aiInsightsResponseSchema`) before display |
| **Fallback** | On any AI error/rate-limit/missing key, a deterministic **rule-based engine** (`lib/insights/rules.ts`) produces equivalent insights, so the feature never breaks |

#### Exact prompt sent to Gemini

The prompt is built deterministically by `lib/insights/prompt.ts`
(`buildInsightsPrompt`). Placeholders in `{…}` are filled from the user's
validated input and the computed footprint:

```text
You are a concise, encouraging climate coach helping someone reduce their personal carbon footprint.
Use the data below to produce specific, realistic, non-judgemental recommendations.

USER FOOTPRINT (estimated):
- Total: {totalTonnes} tonnes CO2e/year
- About {vsSustainableTarget}x the sustainable per-capita target.
Breakdown by category:
- {Category}: {annualKg} kg CO2e/year ({share}% of total)
  …one line per category…

LIFESTYLE DETAILS:
- Car: {carType}, {carKmPerWeek} km/week
- Public transport: {busKmPerWeek} km bus, {trainKmPerWeek} km train per week
- Flying: {flightHoursPerYear} hours/year
- Electricity: {electricityKwhPerMonth} kWh/month ({renewablePercent}% renewable)
- Gas: {gasKwhPerMonth} kWh/month
- Diet: {dietLabel}
- Goods spend: ${monthlySpendUsd}/month

RESPOND WITH STRICT JSON ONLY (no markdown, no commentary) matching this shape:
{
  "summary": "one or two encouraging sentences about their footprint",
  "insights": [
    {
      "category": "transport | energy | diet | goods",
      "title": "short action title",
      "description": "one or two sentences of practical guidance",
      "estimatedAnnualSavingKg": number,
      "difficulty": "easy | medium | ambitious"
    }
  ]
}

Provide 3 to 5 insights, prioritising the largest categories. Keep savings estimates realistic and grounded in the breakdown above.
```

Why this design:
- **Strict-JSON instruction + `responseMimeType`** makes the output reliably
  parseable.
- **Grounding the model in the computed breakdown** keeps savings estimates
  realistic and prevents hallucinated numbers.
- **Schema validation after the call** means malformed AI output degrades
  gracefully to the rule-based fallback rather than breaking the UI.

## Privacy & security notes

- No personal data leaves the browser except the anonymous lifestyle figures
  sent to the insights API (and onward to Gemini) when you request insights.
- The API validates and bounds all input, rate-limits by IP, and never returns
  internal error details to the client.
- A strict CSP and standard security headers are applied to every response (see
  `next.config.mjs`).
